Lecture 12 : Méthode GET

Développement d’API REST : Construire, Tester, Dépanner

La méthode GET est l’une des méthodes HTTP les plus utilisées dans le développement d’API REST. Elle est utilisée pour récupérer des données depuis un serveur sans modifier la ressource sous-jacente.

Lorsqu’un client envoie une requête GET, le serveur traite la requête et renvoie les données demandées, généralement formatées en JSON.

Objectif de la Méthode GET

L’objectif principal de la méthode GET est de récupérer des informations depuis une API REST. Il s’agit d’une opération en lecture seule, ce qui signifie qu’elle ne crée, ne met à jour et ne supprime aucune donnée.

Exemples courants :

  • Récupérer une liste de produits
  • Obtenir les détails d’un client spécifique
  • Récupérer des commandes depuis une base de données

Exemple de Requête GET

Une requête GET typique peut ressembler à ceci :

GET /api/products

Cette requête demande au serveur de renvoyer une liste de tous les produits.

Exemple de Réponse GET

Si la requête est réussie, le serveur peut renvoyer une réponse JSON comme celle-ci :

[ { "id": 1, "name": "Laptop", "price": 1200 }, { "id": 2, "name": "Keyboard", "price": 45 } ]

Cette réponse contient des données structurées que l’application cliente peut traiter.

Requête GET pour une Ressource Spécifique

Une requête GET peut également récupérer une seule ressource en spécifiant son identifiant.

GET /api/products/1

Dans ce cas, le serveur renvoie uniquement le produit avec l’ID 1.

Caractéristiques de la Méthode GET

  • Utilisée pour récupérer des données
  • Ne modifie pas les ressources du serveur
  • Peut être mise en cache par les navigateurs
  • Doit être sûre et idempotente

Tester les Requêtes GET

Les développeurs testent souvent les requêtes GET à l’aide d’outils comme Postman ou un navigateur web. Ces outils permettent d’envoyer des requêtes directement aux endpoints de l’API et de visualiser les réponses.

Exemple d’endpoint testé dans Postman :

https://localhost:5001/api/products

Bonnes Pratiques

  • Utiliser des noms d’endpoints clairs et descriptifs
  • Renvoyer des réponses JSON structurées
  • Utiliser les codes de statut HTTP appropriés
  • Optimiser les requêtes pour de meilleures performances

Prochaines Étapes

Après avoir compris la méthode GET, l’étape suivante consiste à implémenter d’autres méthodes HTTP telles que POST, PUT et DELETE afin de permettre une interaction complète avec les ressources de l’API.
